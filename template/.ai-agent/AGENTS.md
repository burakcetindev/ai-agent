# Coding Agent Operating Rules

## Development Workflow
1. Write modular code
2. Add docstrings
3. Create tests
4. Dry run application
5. Update README
6. Update docs folder
7. Create CHANGELOG_PENDING.md
8. Ask user for commit
9. Delete changelog
10. Commit using conventional commits

## Code Rules
- OOP
- SOLID
- DRY
- Modular
- Extensible

## Testing Rules
- No mocks
- Real verification
- Integration preferred
- Remove obsolete or low-value tests

## Documentation Rules
README must be updated before push

## Architecture Rules
docs/architecture.md required
docs/data-flow.md required
docs/considerations.md required

## Diagrams
Mermaid diagrams mandatory where applicable

## Scripts
scripts/build.sh required
scripts/start.sh required

## Commit Rules
Conventional commits mandatory
CHANGELOG_PENDING.md workflow mandatory
